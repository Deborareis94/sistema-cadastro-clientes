# Integração de Dados e Gestão de Clientes via API

## Sobre o Projeto
Este projeto demonstra a construção de um fluxo de dados completo, unindo uma interface web ao processamento de informações via **API REST**. O foco principal foi criar um sistema capaz de enviar e buscar dados de forma assíncrona, garantindo que o cadastro de clientes seja persistido com segurança em um banco de dados.

## Tecnologias e Integração
- **Frontend (HTML/CSS/JS):** Interface para entrada e visualização de dados.
- **Comunicação (API REST):** Utilização de métodos HTTP (POST, GET, PUT, DELETE) para integrar o frontend ao servidor.
- **Backend (Node.js):** Processamento das requisições da API.
- **Banco de Dados:** Armazenamento estruturado para consultas inteligentes.

##  Diferenciais Técnicos
- **Consumo de API:** O sistema não apenas salva dados, mas se comunica com um servidor para gerenciar as informações de forma dinâmica.
- **Sincronização em Tempo Real:** Uso de requisições assíncronas para atualizar a base de dados sem necessidade de recarregar a página.
- **Estruturação SQL:** Dados salvos prontos para serem extraídos para relatórios de BI.

##  O Valor para Análise de Dados
Dominar o uso de **APIs** é essencial para qualquer analista moderno. Este projeto prova que eu consigo:
1. **Conectar sistemas:** Extrair e enviar dados entre diferentes camadas.
2. **Automatizar a coleta:** Criar o caminho para que o dado saia da mão do usuário e chegue pronto para a análise.
3. **Manter a integridade:** Garantir que a API valide o dado antes de salvá-lo no banco.
